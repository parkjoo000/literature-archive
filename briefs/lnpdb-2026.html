Lipid Nanoparticle Database towards structure-function modeling and data-driven design for nucleic acid delivery
Nature Communications · 2026. 01. 28 · Collins E*, Ji J*, Kim S-G* … Anderson DG†, Im W† · Open access · MIT License
Key findings
01
Database
19,528개 LNP 제형 통합 — LNP 분야의 Protein Data Bank 최초 구축
단일 연구실·조건에서 산출된 LNP structure-function 데이터가 파편화·비표준화 상태로 문헌에 분산되어 있던 문제를 해소하기 위해, 19,528개 제형의 구조 및 기능 데이터를 단일 저장소 LNPDB로 통합. 기존 LiON 학습 원본 데이터셋 대비 규모를 대폭 확대했으며, 신규 제형은 이온화 지질 화학적 다양성 및 4성분(ionizable lipid / helper lipid / cholesterol / PEG-lipid) 몰비 variation을 집중 보강하여 화학 공간(chemical space) 커버리지를 실질적으로 넓힘.
웹 인터페이스: lnpdb.molcube.com — 이온화 지질 구조, helper lipid 종류, molar ratio, 전달 효율(in vitro/in vivo) 기준 검색·필터링·CSV 다운로드 지원. Community contribution 시스템으로 신규 데이터 등록 가능하여 DB의 지속적 확장 구조 내재화.

Featurization 스키마 주요 항목: 이온화 지질 SMILES + 구조 분류, helper lipid 종류(DOPE / DSPC 등), cholesterol·PEG-lipid 몰비, N/P ratio, 핵산 종류(mRNA / siRNA / pDNA), 입자 특성(size / PDI / ζ-potential), 세포주·동물 종·표적 기관, transfection readout(luciferase RLU / GFP MFI 등 다양한 assay 포함). 각 지질의 CHARMM36 FF 파일도 함께 제공.
02
ML / AI
LiON 재학습 — 7개 독립 데이터셋 중 5개에서 Spearman ρ 향상
LiON(Lipid-Informed Optimization Network, Witten et al. Nat Biotechnol 2024)은 이온화 지질 SMILES + 4성분 몰비를 입력으로 LNP delivery efficacy를 예측하는 graph neural network(GNN, chemprop 기반 MPNN) 모델. LNPDB로 재학습한 결과, amine-based 70-15-15% 단일 split 기준 Spearman ρ가 7개 독립 평가 데이터셋 중 5개에서 유의하게 향상.
학습 설정 상세: Python 3.8 + chemprop 1.7.0 환경. LNPDB.csv → /single_split(70-15-15% train-val-test) 및 5개 /cv_splits(80-20% train-test)로 분할 제공. GitHub repo에 trained checkpoint가 포함되어 있어 추가 학습 없이 즉시 inference 가능. 학습 커맨드는 chemprop_train CLI로 재현 가능하며 README에 전체 명령어 수록.

AGILE 비교: 동일 LNPDB heldout set에 대해 AGILE 모델과 성능 비교 수행. LiON embedding과 AGILE embedding 간 overlap 분석으로 shared structure-function pattern 확인 — 데이터 이질성에도 불구하고 두 모델이 유사한 화학적 표현을 학습했음을 시사.
03
MD Simulation
CHARMM36 FF 자동 생성 — LNP 파이프라인에 physics-based 모달리티 최초 도입
기존 LiON 학습 데이터셋에는 MD-ready force field 파일이 전혀 없었음. LNPDB는 등록된 모든 구성 지질에 대해 CHARMM36 topology(.rtf) 및 parameter(.prm) 파일을 자동 생성하여 DB에 포함. 이는 실험 데이터 전용 파이프라인에 physics-based 구조 분석 경로를 추가하는 구조적 전환을 의미하며, GROMACS / NAMD 등 주요 MD 엔진에서 바로 사용 가능.
시뮬레이션 구성: 중성(neutral, pH 7.4) 및 양성자화(protonated, pH ~5, 내소체 조건 모사) 상태 각각 all-atom bilayer patch 시뮬레이션 수행. Trajectory 파일은 용량 문제로 GitHub에 직접 미포함 → Dropbox 링크로 representative trajectory 및 ini/last snapshot 공개.

분석 스크립트: analyze_CPP_Rg.ipynb (radius of gyration 기반 CPP), analyze_CPP_V.ipynb (체적 기반 CPP), analyze_other_metrics.ipynb (bilayer stability 외 추가 지표). Python 3.9 + lnpdb conda 환경으로 재현 가능. 모든 결과는 /results 폴더에 사전 계산값 포함.
04
MD Simulation
CPP·Bilayer stability가 delivery 성능의 구조적 예측 지표로 유효함을 DB 규모에서 정량화
MD 시뮬레이션으로 산출한 critical packing parameter(CPP) 및 bilayer stability가 실험적 LNP delivery 성능과 유의미한 상관관계를 보임을 LNPDB 스케일에서 최초로 정량화. CPP는 lipid 기하학적 shape(head area, tail volume, tail length)를 수치화한 지표로, CPP > 1(cone-inverted)일수록 역육방형(Hₙ) 구조로 전이되기 쉬워 endosomal escape 효율이 높다는 기존 가설을 DB 규모로 검증.
Bilayer stability 해석: protonated 조건(산성 pH, 내소체 환경 모사)에서 bilayer가 불안정해지는 정도 → endosomal escape 능력과 정(+)의 상관. Neutral 조건에서는 안정적이어야 혈중 안정성이 유지됨. 두 조건 간 stability 차이(Δstability)가 선택적 내소체 불안정화의 통합 지표로 제안됨.

설계 함의: CPP·bilayer stability를 in silico pre-screening 지표로 사용하면 실험 대상 후보군 축소 및 합리화 설계 가능. 단, 본 연구는 상관관계 분석 수준이며, prospective 검증(예측 → 실험 확인 루프)은 포함되지 않음.
05
LNP formulation
개방형 community contribution 구조 — 분야 인프라로서의 지속 확장성 설계
단순 데이터 공개를 넘어, 연구자들이 자체 LNP 데이터를 표준 형식으로 등록·공유할 수 있는 community contribution 시스템을 구현. PDB가 단백질 구조 데이터의 공용 인프라로 발전한 성장 경로를 LNP 분야에 적용하려는 설계 의도가 명시적으로 표현됨. 현재 DB는 수동 큐레이션 중심이나, 외부 기여를 통한 자동 확장 구조를 목표로 함.
현 DB 스냅샷 특성: in vitro 데이터 비중이 높으며 간 세포주(Huh7, HepG2, AML12) 또는 HeLa 기반 transfection efficiency가 주요 readout. mRNA·siRNA·pDNA 모두 포함되나 mRNA 기반 제형의 비중 상대적으로 높음. 이온화 지질 pKa 범위는 대체로 6.2–6.8에 집중(간 targeting 최적 범위). Non-liver 표적 데이터는 상대적으로 희소.
Methods overview
DB 구축
문헌 19,528개 LNP 제형 수동 큐레이션 + 표준 featurization. 이온화 지질 SMILES, 4성분 몰비, 핵산 종류, 세포·장기 대상, transfection readout 통일 인코딩. CHARMM36 FF 파일 자동 생성 포함.
딥러닝 (LiON)
Framework: chemprop 1.7.0 (MPNN). 입력: 이온화 지질 molecular graph + 제형 수치 feature. 평가: Spearman ρ. Split: 70-15-15% single + 5-fold CV (80-20%). Python 3.8.
MD Simulation
FF: CHARMM36. 조건: neutral(pH 7.4) + protonated(pH ~5) bilayer patch. 지표: CPP(Rg/V 기반), bilayer stability(Δ between conditions). Python 3.9 분석 스크립트.
비교 / 검증
AGILE 모델과 동일 heldout set 비교. LiON–AGILE embedding overlap 분석. Spearman ρ 기준 7개 독립 데이터셋에서 원본 LiON과 성능 비교.
Limitations
—
이종 데이터셋 통합의 배치 효과 미해결
19,528개 제형이 다수 독립 연구실·프로토콜에서 수집됨에 따라 세포 seeding density, mRNA 용량, 측정 시점, 기기 간 차이로 인한 체계적 배치 효과가 잔존. 논문 Methods에서 이를 명시하나 보정 방법론은 제시되지 않아, 통합 데이터로 훈련한 모델의 예측 편향이 데이터 출처에 따라 가변적일 수 있음.
—
LiON 성능 개선의 비일관성 — 2개 데이터셋에서 개선 없음
7개 평가 데이터셋 중 2개에서는 원본 LiON과 동등하거나 열등. 개선되지 않은 데이터셋의 지질 화학적 특성 또는 측정 조건 차이가 원인으로 추정되나 세부 분석이 제한적. 특정 이온화 지질 클래스 또는 비-amine계 지질로의 일반화 한계 가능성을 시사.
—
MD 시뮬레이션 — 단일 trajectory, 앙상블 통계 및 실제 LNP 구조 미반영
각 지질 시스템에 대해 representative trajectory 1개 수준으로 CPP·stability 산출 — 앙상블 평균 없이 통계적 신뢰구간 불명확, 초기 구조 의존성 문제 상존. 더불어 단분자 bilayer patch 수준의 시뮬레이션으로, 실제 LNP 내부의 역육방형 지질 복합체 구조 및 핵산 포획 상태, 다른 LNP 구성 성분(cholesterol, PEG-lipid 등)과의 상호작용이 반영되지 않음.
—
In vitro 데이터 편중 — in vivo 상관성 및 비간 표적 검증 부재
DB 내 데이터 대부분이 세포 수준(in vitro) transfection efficiency 기준이며, 간 이외 장기 targeting, 면역원성, PK/PD 파라미터는 포함되지 않음. 이온화 지질 pKa가 간 targeting 최적 범위(6.2–6.8)에 집중되어 있어, 폐·비장·T세포 등 non-liver 표적 LNP 설계 시 DB 및 LiON 예측값의 직접 적용 가능성은 별도 검증 필요.
—
Community contribution 품질 관리 체계 미공개
외부 연구자의 데이터 기여를 허용하는 구조이나, curation 자동화 수준·QC 기준·버전 관리 정책이 현재 공개되지 않음. 향후 DB 확장 시 훈련 데이터 오염(noisy label) 위험이 상존하며, 기여 데이터와 기존 데이터 간 가중치 처리 및 중복 제거 방식도 불명확.
—
CPP·stability 지표의 인과성 미확립 — prospective 검증 실험 부재
CPP와 bilayer stability가 delivery 성능과 상관관계를 보임은 정량화됐으나, 이 지표가 성능을 인과적으로 결정하는지는 검증되지 않음. 특히 in silico CPP 기반 사전 스크리닝이 실험 적중률(hit rate)을 실제로 높이는지에 대한 전향적(prospective) 검증 실험이 미포함 — 실용적 스크리닝 도구로서의 실증이 부족.
실무 관점 (DDS 연구자 시각)
즉시 활용 가능한 리소스: lnpdb.molcube.com에서 이온화 지질 구조·제형 조성·전달 효율을 교차 필터링해 CSV 다운로드 가능. 자체 스크리닝 데이터와 병합 시 LiON 재학습 또는 통계 모델 구축에 바로 투입 가능한 수준. CHARMM FF 파일 포함으로 GROMACS/NAMD 기반 MD 분석 진입 장벽이 크게 낮아짐.
LiON 재학습 파이프라인: GitHub repo에 LNPDB.csv + single/cv split + trained checkpoint 모두 포함. chemprop 1.7.0 환경 세팅 후 즉시 재현 가능. 자체 제형 데이터(수십~수백 개) 추가 후 fine-tuning 또는 transfer learning 적용 시, 소규모 실험 데이터만으로 예측 모델 구축 가능한 현실적 파이프라인.
MD 활용 전략: 전체 앙상블보다는 lead 후보군(3–5개) 간 상대적 CPP·stability 비교 목적으로 활용하는 것이 현실적. 동일 프로토콜로 비교하면 단일 trajectory라도 상대적 ranking에 유용. 새 이온화 지질 설계 시 CHARMM FF 자동 생성 기능으로 parametrization 시간 대폭 단축 가능.
국내 협업 접점: KIST 계열(Gyochang Keum, Eun-Kyoung Bang), ASAN Medical Center(Jun-O Jin) 공저자 포함. Community contribution 참여로 자체 제형 데이터 DB 등록 및 공동 활용 가능. MIT Anderson Lab(Langer/Anderson) 협업 채널 간접 접근점.
적용 범위 주의: DB의 transfection readout 대부분이 간 세포 기반 in vitro 데이터이며 pKa 범위가 간 targeting 편향. 비간(non-liver) LNP 설계 시 LiON 예측값 신뢰도는 별도 검증 필요. 또한 mRNA 용량·세포 seeding density 등 측정 조건 차이가 readout에 상당한 영향을 미칠 수 있으므로, 자체 데이터와 병합 시 정규화 전처리 필수.
